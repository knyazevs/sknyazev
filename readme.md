# knyazevs

Персональный архитектурный профиль — альтернатива резюме для инженера,
чья основная ценность в проектировании систем, а не в списке технологий.

## Идея

Классическое резюме плохо передаёт архитектурное мышление.
Этот проект построен иначе: сам сайт является доказательством компетентности —
через структуру репозитория, задокументированные решения и работающую систему.

Репозиторий читается как проект: есть домен, есть ADR, есть реализация.

## Структура

```
docs/          — источник истины (читается напрямую из репо)
  adr/         — архитектурные решения
  diagrams/    — C4-диаграммы
  profile/     — профиль и образование
  experience/  — опыт работы
  skills/      — компетенции
  projects/    — проекты

app/           — исходный код сайта (Astro + Svelte)
```

## Как это работает

Сайт разделён на две части:

- **Левая** — документация из этого репозитория в реальном времени.
  В продакшене читается через GitHub API, в dev — через локальную файловую систему.

- **Правая** — голосовой интерфейс на базе LLM, который отвечает на вопросы
  строго на основе левой части. Не свободный чат — интерпретатор архитектурного замысла.

## Запуск локально

```sh
cd app
bun install
bun run dev
```

## Технологии

Astro, Svelte 5, Bun — фронтенд.  
Деплой: GitHub Pages + GitHub Actions.  
LLM: OpenRouter. Голос: OpenAI TTS + Whisper.

Выбор технологий зафиксирован в [`docs/adr/`](docs/adr/).
