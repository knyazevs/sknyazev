# knyazevs

Персональный архитектурный профиль — альтернатива резюме для инженера,
чья основная ценность в проектировании систем, а не в списке технологий.

## Идея

Классическое резюме плохо передаёт архитектурное мышление.
Этот проект построен иначе: сам сайт является доказательством компетентности —
через структуру репозитория, задокументированные решения и работающую систему.

Работодатель видит не только «что сделано», но и человека:
как мыслит, что выбирает и зачем.

## Архитектура

Три слоя, каждый верифицируем независимо:

**Контентный** (`docs/`) — ADR, блог, профиль, опыт. Читается напрямую из репозитория.

**Визуальный** (`app/`) — Astro + Svelte 5, WebGL-орб, чат-интерфейс.
Нет split-screen: орб по центру, документы открываются через модальное окно.

**Интеллектуальный** (`server/`) — Kotlin/Ktor CIO, RAG-пайплайн поверх всего репозитория.
Агент отвечает строго на основе документации и исходного кода — не выдумывает.

## Структура

```
docs/
  adr/            — архитектурные решения (ADR-019..N)
  blog/           — записи от первого лица, живые нарративы
  profile/        — профиль автора, образование, принципы
  experience/     — опыт работы с cv_description для PDF
  skills/         — компетенции
  projects/       — проекты

app/              — Astro + Svelte 5 (фронтенд)
server/           — Kotlin/Ktor + RAG-пайплайн (бэкенд)
scripts/          — build-cv.ts (генератор PDF-резюме)
```

## RAG-пайплайн

Агент индексирует весь репозиторий:
- `DocumentLoader` — `docs/` рекурсивно, чанкует по H2/H3
- `CodeLoader` — `server/src/`, `app/src/`, `scripts/`, конфиги; чанкует по объявлениям функций/классов

Поиск: HyDE (гипотетический ответ) + BM25 + RRF-слияние → top-5 чанков → LLM.
После ответа агент эмитирует источники — пользователь видит кликабельные чипы.

## Запуск локально

**Бэкенд:**
```sh
cd server
cp .env.example .env   # заполнить OPENROUTER_API_KEY и др.
./gradlew run
```

**Фронтенд:**
```sh
cd app
bun install
bun run dev
```

**PDF-резюме:**
```sh
bun run build:cv   # → app/public/cv.pdf
```

## Технологии

**Frontend:** Astro 6, Svelte 5, WebGL2, TypeScript  
**Backend:** Kotlin, Ktor CIO, Exposed, PostgreSQL (опционально), OpenRouter  
**Voice:** OpenAI TTS + Whisper  
**CV:** Node 22 + Puppeteer  
**Deploy:** GitHub Pages + GitHub Actions

Выбор технологий зафиксирован в [`docs/adr/`](docs/adr/).
