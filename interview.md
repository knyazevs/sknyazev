# Интервью-подготовка — кэш обсуждения

> Рабочий документ для продолжения дизайна в новой сессии. Не финальная документация.
> Дата обсуждения: 2026-04-19

---

## Цель

Подготовка к интервью на Technical Lead / Architect / Staff+.
Нужно: документировать вопросы с развёрнутыми ответами + система тренировки с повторениями.
Артефакт должен быть публично верифицируемым (в духе CLAUDE.md) — часть CV-проекта.

## Выбранный вариант: B (контент + веб-режим тренировки)

Рассматривались:
- **A** — только контент в markdown (мало, не показывает процесс).
- **B** ✅ — контент + страница `/interview` с тренажёром.
- **C** — CLI-quiz (приватно, не даёт публичной ценности).

Причина выбора B: попадает в цель проекта (публичный артефакт) + переиспользует RAG.

## Трёхслойная структура контента

Читаю → тренируюсь → закрепляю.

```
docs/interview/
  01-system-design/
    README.md          ← входная точка, оглавление категории
    theory.md          ← конспект для чтения (20-40 мин, 500-1500 слов)

    resources.md       ← книги, статьи, видео + «чего НЕ читать»
  02-architecture/
  ...
```

Однострочный кор.

### Follow-up вопросы
- ...

## Шаблон `theory.md` (конспект, не учебник)

```markdown
## Карта темы
Структура: из чего тема состоит.

## Ключевые концепции
### <Концепция>
2-3 абзаца + пример из практики/ADR.

## Частые трейдоффы
Таблица: ситуация → варианты → что выбрать когда.

## Что любят спрашивать
Список тем-триггеров без ответов.
```

Тон — конспект от первого лица для повторения уже известного.

## UI `/interview` — 4 режима

1. **Learn** — чтение `theory.md` по выбранной категории, в конце кнопка «Проверить знания» → переход в Train по этой категории.
2. **Train** — случайный вопрос → таймер 30 сек (короткий ответ) → таймер 2 мин (развёрнутый) → опционально textarea → «показать эталон» → самооценка 1-5 → следующий.
3. **Review** — список всех вопросов с фильтрами по категории/сложности.
4. **Stats** — heatmap, что давно не видел, распределение по Leitner-коробкам.

Sidebar в Train-режиме — опциональный cheatsheet текущей категории.

## Spaced repetition: Leitner box

5 коробок, интервалы 1 / 3 / 7 / 14 / 30 дней. Проще SM-2, достаточно для интервью.

Хранение: `localStorage` ключ `interview:progress`:
```ts
{ [questionId]: { box: 1-5, lastSeen: ISOdate, history: Array<1-5> } }
```

Экспорт/импорт JSON — для переноса между устройствами без бэкенда.

## Интеграция с RAG

- **Индексировать:** `theory.md`, `questions.md` (секции «Ответ» + «Развёрнутый»), `resources.md`.
- **НЕ индексировать:** `cheatsheet.md` (шумит поиск), «Типичные ошибки» / «чего не знаю» (могут выглядеть странно в цитате агента).

Реализация фильтрации секций — открытый вопрос (через спец-маркер H2 или отдельный файл).

## Связь теория ↔ вопросы

- В вопросе `Refs: theory.md#anchor` — клик ведёт к разделу теории.
- В конце раздела теории — список связанных вопросов.
- Это граф, который работает и в UI, и в RAG.
