# Архитектура систем

## Что проектировал

- **Low-code платформа** — система, которая строит другие системы.
  Архитектурные решения с мультипликативным эффектом на всю экосистему продуктов
- **SaaS-платформа для коллаборации** — интеграция концептуально разных подсистем
  (видео, чат, документы, доска) в единый продукт
- **Платёжные коннекторы** — интеграции с банковскими системами в финтехе
- **B2B-продукты** — от greenfield до стабилизации legacy

## Подходы

- Architecture Decision Records (ADR) как инструмент фиксации решений и их контекста
- C4-модель для документирования архитектуры
- Осознанные компромиссы: знаю, когда монолит лучше микросервисов
- Миграции без остановки: рефакторинг и смена стека в работающих системах

## Где сильнее всего

Архитектура в условиях ограничений и неопределённости:
когда нет идеального решения и нужно выбрать наименее плохое,
понимая последствия каждого варианта.

Диагностика и стабилизация систем, выросших из MVP.
