# Техническое лидерство

## Роли

- **Руководитель backend-команды** (7–13 человек) — полная ответственность
  за архитектурные и технические решения
- **Tech lead** — проектирование, код-ревью, технические стандарты
- **R&D-лид** — исследование и внедрение новых технологий в продукт
- **Преподаватель** — современные методы разработки, проектирование
  и архитектура, университет (2022–2025)

## Как работаю с командой

Влияние через аргументы и структуру, а не через статус.
Решения обосновываю, а не спускаю сверху.

Лучший результат — когда команда понимает *почему* принято то или иное решение,
а не просто выполняет задачу.

## R&D как практика

Систематический поиск технологий по двум осям:
- улучшение качества продукта
- улучшение качества работы разработчиков (DX)

Примеры внедрений: Kotlin и KMP, переход на GitHub с его инфраструктурой,
AI-инструменты в разработку — всё до того, как стало очевидным.
