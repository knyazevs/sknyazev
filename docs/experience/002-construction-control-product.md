# ПК СтройКонтроль — Цифровая трансформация строительного контроля

## Контекст системы

B2B-продукт для полноценной цифровизации строительного контроля:
замечания, инспекции, чек-листы, мобильное приложение для работы на объектах.
Интеграции со сторонними строительными и проектными системами.

Стек: Java, Gradle, Hibernate, Redis, React, React Native, Jenkins.

К моменту прихода — уже зрелый продукт, переданный на поддержку.
Впоследствии заморожен: только критичные баги.

## Контекст роли

Первый продукт в компании. Команда (7–13 человек) получила СК на поддержку
как устаревший продукт — стандартный онбординг для новых участников.
Впоследствии вся команда переключена на МРС Платформу.

## Ключевое решение: миграция MongoDB → PostgreSQL

По инициативе автора принято и реализовано решение о переходе
с MongoDB на PostgreSQL.

**Контекст:** реляционная природа данных строительного контроля
плохо ложилась на документоориентированную модель.
Дополнительная мотивация — снижение операционной нагрузки.

**Результаты:**
- Упрощены обновления безопасности (единый стек БД)
- Ускорена часть запросов
- Устранена необходимость хостить и обслуживать MongoDB
- Резервное копирование и доступность переданы облачному провайдеру (Selectel)

Оценить бизнес-эффект полностью не удалось — продукт заморозили
вскоре после миграции, что само по себе подтверждает
правильность решения: снижение стоимости владения legacy-системой.
