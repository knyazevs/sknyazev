# Персональный архитектурный профиль — этот сайт

## Что это

Альтернатива резюме для инженера, чья ценность — в архитектурном мышлении,
а не в списке технологий. Сайт спроектирован как production-grade система:
с ADR, доменной моделью, задокументированными компромиссами.

Сам сайт является доказательством компетентности — не описывает подход, а демонстрирует его.

## Архитектурная идея

Одна система. Два представления.

**Левая часть** — документация из этого репозитория в реальном времени.
Источник истины. Читается через GitHub API в продакшене,
через локальную файловую систему в dev.

**Правая часть** — голосовой интерфейс на базе LLM.
Отвечает на вопросы строго на основе левой части.
Не свободный чат — интерпретатор архитектурного замысла.

## Ключевые решения

Все решения задокументированы в [`docs/adr/`](../adr/) —
14 ADR от доменной модели до CI/CD.

Наиболее нетривиальные:
- **Dual-source docs** (ADR-012): dev читает локально, prod — GitHub API в реальном времени
- **LLM с жёсткими ограничениями** (ADR-003): запрещено придумывать факты,
  каждый ответ трассируем к источнику
- **Astro + Svelte islands** (ADR-009): статика там где не нужна реактивность,
  острова там где нужна

## Стек

| Слой | Технология |
|------|-----------|
| Frontend | Astro 6, Svelte 5 |
| Runtime | Bun |
| LLM | OpenRouter (vendor-agnostic) |
| TTS / ASR | OpenAI TTS + Whisper |
| Деплой | GitHub Pages + GitHub Actions |

## Намеренные не-цели

- Не оптимизируется под рекрутеров и ATS
- Не универсальный AI-чат
- Визуальные эффекты только там, где несут смысл

## Что демонстрирует

Репозиторий читается как инженерный проект: есть домен, есть решения,
есть реализация. Человек, понимающий системы, увидит здесь
не портфолио, а рабочий артефакт.
