# C4 L1 — Context: Персональный архитектурный профиль (сайт)

> Уровень: C4 / L1 (System Context)  
> Цель: показать границу системы и её взаимодействия с внешними акторами/системами.  
> Нотация: Mermaid (C4-like)

```mermaid
flowchart LR
  %% Акторы
  user["Пользователь<br/>(CTO / Principal / инженер)"]
  browser["Клиент — браузер<br/>(документация + интерактив)"]

  user -->|"вопросы, навигация"| browser

  %% Граница системы
  subgraph system["Система: Персональный архитектурный профиль"]
    direction LR

    subgraph left["Левая часть: Источник истины"]
      docs["Документация<br/>(ADR, диаграммы, описания)"]
    end

    subgraph right["Правая часть: Интерактивное представление"]
      orchestrator["Оркестрация<br/>(состояния / события)"]
      llm["LLM интерпретатор"]
      voice["Голосовой слой<br/>(ввод / вывод)"]
      captions["Субтитры"]
      visual["Абстрактная сущность<br/>(визуализация)"]
    end
  end

  %% Внешние зависимости
  mic["Микрофон"]
  audio["Аудиовывод"]

  %% Связи
  browser <-->|"чтение / навигация"| docs
  browser -->|"ввод, команды"| orchestrator
  docs -->|"контекст"| orchestrator

  orchestrator -->|"структурированный контекст"| llm
  llm -->|"текст ответа"| orchestrator

  orchestrator -->|"текст"| captions
  captions -->|"отображение"| browser

  orchestrator -->|"команды"| voice
  voice -->|"воспроизведение"| audio

  mic -->|"аудиоввод"| voice
  voice -->|"текст запроса"| orchestrator
  voice -->|"сигнал речи"| visual
  visual -->|"отрисовка"| browser
```

## Примечания к чтению диаграммы

- **Граница системы** включает две концептуальные части:  
  **Источник истины** (лево) и **интерактивное представление** (право).
- **Канонический контент** в интерактивной части — текст (вопрос/ответ); голос и визуализация — производные представления.
- Внешние зависимости показаны намеренно минимально (устройства ввода/вывода). Если появится внешний провайдер LLM/TTS/ASR — он будет добавлен на L2 (Container) как внешний контейнер/сервис.
