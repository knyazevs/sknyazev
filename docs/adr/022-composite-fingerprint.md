# ADR-22: Составной fingerprint для rate limiting

- Статус: Принято
- Дата: 2026-04-14
- Авторы: Sergey Knyazev, Claude Sonnet 4.6
- Теги: security, fingerprint, rate-limit, nginx, tls, ja3, canvas

## Контекст

ADR-21 ввёл rate limiting по IP-адресу. IP — слабый идентификатор:

- **VPN/прокси**: IP меняется за секунды, лимит сбрасывается
- **NAT/CGNAT**: несколько пользователей делят один IP → ложные срабатывания
- **Мобильные сети**: IP меняется при смене базовой станции

Необходим идентификатор, который остаётся стабильным при смене IP
и одновременно не поддаётся тривиальной подделке.

---

## Решение

Заменить IP-ключ на **составной fingerprint** из трёх независимых сигналов,
каждый из которых труден для обхода самостоятельно.

### Архитектура

```
Browser ──[X-Fingerprint: <canvas-sha256>]──► Nginx
                                               │ добавляет X-JA3-Fingerprint, X-Real-IP
                                               ▼
                                             Ktor
                                               │ SHA256(ja3 + clientFP + ip) → ключ rate limit
                                               ▼
                                           RateLimit
```

### Сигнал 1: Canvas fingerprint (клиентская сторона)

Браузер рендерит Canvas-сцену (fillRect + fillText) и вычисляет SHA-256
от `toDataURL()` вместе с browser signals:

```
components = [
  navigator.userAgent, navigator.language, navigator.hardwareConcurrency,
  screen.width, screen.height, screen.colorDepth, devicePixelRatio,
  Intl timezone,
  canvas.toDataURL()          ← GPU/драйвер/шрифты — уникально на уровне железа
]
SHA-256(components.join("|||")) → X-Fingerprint: <hex64>
```

Вычисляется один раз при загрузке страницы, кэшируется в модульной переменной.
Передаётся в каждом запросе как заголовок `X-Fingerprint`.

Реализация: `app/src/lib/fingerprint.ts` (~40 строк TS).
Без внешних библиотек — только Web Crypto API.

### Сигнал 2: JA3 TLS fingerprint (серверная сторона)

JA3 — хэш параметров TLS ClientHello:
`cipher_suites + tls_extensions + elliptic_curves + elliptic_curve_point_formats`

Определяется TLS-библиотекой браузера (BoringSSL в Chrome, NSS в Firefox).
Каждый браузер имеет стабильную уникальную JA3-сигнатуру.
Подделать = перекомпилировать браузер или переписать TLS-стек.

Реализация: Nginx собирается с модулем `nginx-ssl-fingerprint`,
который добавляет переменную `$ssl_ja3`.
Nginx передаёт значение в Ktor через `proxy_set_header X-JA3-Fingerprint $ssl_ja3`.

### Сигнал 3: IP-адрес (дополнительный)

IP остаётся как третий сигнал — слабый сам по себе,
но в сочетании с fingerprint создаёт полный составной ключ.

### Составной ключ

```kotlin
val key = sha256("$ja3:$clientFingerprint:$ip")
```

Каждый компонент можно подделать по отдельности (IP — VPN, fingerprint — скрипт).
Но для обхода нужно подделать все три одновременно — практически неосуществимо
в контексте случайного злоупотребления.

---

## Обоснование

### Почему canvas, а не FingerprintJS

FingerprintJS — готовая библиотека с высокой точностью (~99.5%).
Отклонено: +22kB bundle, чёрный ящик, не демонстрирует понимание механики.

Canvas fingerprint вручную (~40 строк) — достаточная точность для задачи,
полный контроль, наглядно демонстрирует принцип работы.

### Почему JA3, а не HTTP-заголовки

HTTP-заголовки (User-Agent, Accept-*) тривиально подделываются любым HTTP-клиентом.
JA3 определяется на уровне TLS — ниже уровня приложения.
Для подделки нужен полный контроль над TLS-стеком браузера.

### Почему nginx, а не TLS-терминация в Ktor

Ktor (JVM) не имеет доступа к raw TLS ClientHello данным.
JA3 требует перехвата на уровне TLS-handshake.
Nginx с кастомным модулем — стандартное решение для этой задачи.
Nginx всё равно нужен как reverse proxy для SSL-терминации.

### Почему три сигнала вместо одного

Ни один сигнал не является абсолютным:
- Canvas: не работает в headless/no-GPU окружениях
- JA3: одинаков для всех пользователей одной версии Chrome на macOS
- IP: меняется при VPN

Комбинация трёх делает обход нетривиальным без полного контроля
над браузером, TLS-стеком и сетевым окружением одновременно.

---

## Последствия

### Положительные
- Rate limit устойчив к смене IP (VPN, мобильные сети)
- JA3 не подделывается без изменения TLS-стека
- Canvas fingerprint уникален на уровне аппаратного GPU
- Fail-safe: при отсутствии заголовков → fallback на IP (деградация, не отказ)

### Отрицательные / Стоимость
- Nginx требует компиляции с кастомным модулем (усложняет Docker build)
- Canvas fingerprint может быть нестабилен в нестандартных браузерах/расширениях
- JA3 одинаков для всех пользователей одной версии браузера + ОС
  (ограничение компенсируется canvas + IP)
- Добавляет 1 заголовок к каждому запросу (~64 байта)
