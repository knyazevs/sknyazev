# ADR-20: Единый модальный контейнер для Документации, Кода и Истории

- Статус: Принято
- Дата: 2026-04-18
- Автор: Sergey Knyazev
- Теги: ui, навигация, единообразие, композиция

## Контекст

На главной странице проекта находится орб и чат. Вспомогательный контент —
документация (`docs/`), код репозитория и история коммитов — до этого момента
открывался тремя разными способами:

1. **Документация** — модальное окно `DocumentModal`, вызываемое из топбара и из
   чипов источников под ответами агента.
2. **Код** — отдельная страница `/code` (`CodeBrowser.svelte`) с полной
   навигацией и sidebar-деревом, открывающаяся по ссылке в топбаре и при клике
   на чип источника вида `code/...`.
3. **История** — отдельная страница `/timeline` с git log, открывающаяся по
   ссылке в топбаре.

Это нарушало принцип единообразия: три разных контекста открывались по-разному,
пользователь то уходил на отдельную страницу, то оставался на главной. Разрыв в
поведении затруднял возврат к диалогу: чтобы продолжить разговор с агентом после
чтения кода, нужно было делать «назад» в браузере.

Одновременно с этим три интерфейса повторяли похожие паттерны (поиск/список,
детальный просмотр, заголовок с закрытием), но имели разную типографику,
отступы и поведение кнопок.

## Решение

Объединить три представления в один компонент `ContentModal.svelte` с
**подчёркнутыми вкладками** в шапке: «Документация» / «Код» / «История».

1. Модалка — единая точка входа для любого вспомогательного контента.
2. Иконки в топбаре главной страницы открывают модалку на нужной вкладке, а не
   навигируют на отдельную страницу.
3. Страницы `/code` и `/timeline` сохраняются как URL-якоря: они рендерят
   главную страницу и немедленно открывают модалку на соответствующей вкладке
   через проп `autoOpenTab`.
4. Клик на чип источника вида `code/...` открывает модалку на вкладке «Код» с
   предварительно выбранным файлом (без перехода по URL).
5. Ширина модалки адаптируется к вкладке: `720px` для документации
   (читабельные строки) и `960px` для кода/истории (дерево + просмотр файла).

### Данные

- **Docs** — как и раньше: в dev через vite-плагин `/api/docs/*`, в prod через
  GitHub Contents API + raw.githubusercontent.
- **Code** — как и раньше: vite-плагин `/api/code/*` + GitHub Git Trees API.
- **Timeline (новое)** — добавлен эндпоинт `/api/commits` в vite-плагине
  (выполняет `git log` в корне репозитория) и fallback на GitHub Commits API в
  prod. Клиент — `getCommits(limit)` в `docs-client.ts`.

## Обоснование решения

- **Единый ментальный контейнер.** Пользователь знает: всё вспомогательное
  открывается «поверх» главной, и Esc возвращает его к диалогу. Это снижает
  когнитивную нагрузку и делает поведение предсказуемым.
- **Сохранение контекста диалога.** Чтение ADR, кода или истории не прерывает
  сессию чата: при закрытии модалки история сообщений остаётся.
- **Переиспользование кода.** Три компонента со схожей механикой (header,
  список, детальный просмотр, закрытие по Esc) свернулись в один, что снижает
  риск расхождения поведений при будущих правках.
- **Подчёркнутые вкладки вместо пилюль/сегментов.** Табы читаются как
  «разделы одного пространства», а не как «разные инструменты». Это лучше
  соответствует тому, что все три вкладки — это разные срезы одного репозитория
  (документация, код, история).

## Последствия

### Позитивные

- Единообразная навигация: все три источника вспомогательного контента
  открываются одним способом и закрываются одной кнопкой/клавишей.
- Прямые URL `/code` и `/timeline` сохраняются (SEO, deep links, внешние
  ссылки), но открывают ту же модалку — один контракт поведения.
- Уменьшилось количество компонентов: `CodeBrowser.svelte` удалён, его
  функциональность — внутри `ContentModal`.
- Чипы источников больше не перекидывают пользователя на другую страницу —
  диалог не теряется.

### Негативные / Компромиссы

- Страницы `/code` и `/timeline` теперь не являются самостоятельными
  полноэкранными представлениями: это главная + автоматически открытая
  модалка. Для потребителей, рассчитывающих именно на полноэкранный просмотр
  кода, это шаг назад.
- Ленивая подгрузка дерева кода и коммитов теперь происходит при первом
  переключении на вкладку. Это осознанный выбор (не тратить сеть зря), но
  первое переключение может занять ~200–500мс.
- Ширина модалки меняется между вкладками (720/960px) — компромисс между
  читабельностью документации и удобством просмотра кода.

## Рассмотренные альтернативы

### Альтернатива A: Оставить как было (3 разных представления)

**Отклонено**: нарушает единообразие и заставляет пользователя запоминать три
разных способа возврата к диалогу.

### Альтернатива B: Сделать всё тремя модалками (отдельные компоненты)

**Отклонено**: решает проблему «уходов на отдельные страницы», но сохраняет
разрозненность — разная разметка, разные отступы, разная анимация открытия.
Один компонент с вкладками даёт строгий единый контракт.

### Альтернатива C: Вкладки как пилюли / сегментированный контрол

**Отклонено**: пилюли визуально сигналят «выбор инструмента из набора». Для
трёх срезов одного пространства лучше подчёркнутые табы — они читаются как
«разделы» и не соперничают за внимание с орбом и чатом.

### Альтернатива D: Удалить страницы `/code` и `/timeline`

**Отклонено**: эти URL уже могут быть в закладках / в индексе поисковиков /
упомянуты в резюме. Редирект на `/?tab=code` — дополнительный прыжок. Проще:
страницы остаются, но рендерят главную с автооткрытой модалкой.
