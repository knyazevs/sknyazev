# ADR-10: Мобильная адаптация лэйаута

- Статус: Принято
- Дата: 2026-04-01
- Авторы: Sergey Knyazev, Claude Sonnet 4.6
- Теги: ui, мобильный, адаптивность, лэйаут

## Контекст

На десктопе сайт использует сплит-лэйаут: две колонки видны одновременно (ADR-7).
На мобильных устройствах ширина экрана не позволяет отображать обе части рядом
без ущерба для читаемости и usability.

Необходимо решить, как адаптировать двухколоночный лэйаут для узких экранов,
сохранив концептуальное разделение двух частей.

---

## Решение

На мобильных устройствах использовать **таб-свитчер**:
две вкладки переключают видимую часть, при этом обе части остаются смонтированными
в DOM (не уничтожаются при переключении).

Вкладки:
- **"Документация"** — левая часть (ADR, о себе, readme)
- **"Интерфейс"** — правая часть (голосовой/текстовый чат)

По умолчанию активна вкладка **"Интерфейс"** — это первое, что видит пользователь,
открывший сайт с телефона.

---

## Обоснование решения

Таб-свитчер сохраняет концептуальное разделение двух частей
и не создаёт ощущения "одной длинной страницы".

Монтирование обеих частей в DOM предотвращает потерю состояния
при переключении: голосовой интерфейс не сбрасывается,
если пользователь временно переключился в документацию.

Выбор "Интерфейс" как стартовой вкладки обусловлен приоритетом wow-эффекта:
первое впечатление важнее структуры документации (ADR-7).

---

## Альтернативы и причины отказа

### Вертикальный стек (обе части друг под другом)
**Отказано**: размывает разделение двух представлений,
создаёт ощущение обычного лэндинга со скроллом.

### Скрытие одной части на мобильном
**Отказано**: нарушает полноту опыта, пользователь теряет доступ к документации или интерфейсу.

### Отдельные URL для каждой части
**Отказано**: усложняет навигацию и разрушает концепцию "одна система, два представления".

---

## Последствия

### Положительные
- Обе части доступны на любом устройстве
- Состояние не сбрасывается при переключении вкладок
- Пользователь сразу видит wow-эффект интерфейса

### Отрицательные / Стоимость
- Одновременная видимость обеих частей (ключевое свойство десктопа) на мобильном недостижима
- Таб-бар занимает часть экрана

---

## Примечание

Брейкпоинт переключения между сплит-лэйаутом и таб-свитчером
определяется на уровне реализации и не является архитектурным решением.
