# Architecture Decision Records

Каждый ADR фиксирует контекст, рассмотренные варианты и обоснование выбора —
включая компромиссы. Нумерация строго последовательная; устаревшие решения не
удаляются, а получают статус «Устарело».

См. `how-to.md` для правил написания нового ADR.
