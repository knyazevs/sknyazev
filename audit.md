## Предложение по H4 — как ограничить что попадает в RAG

Текущее состояние: `CodeLoader` индексирует `CLAUDE.md` (мета-инструкции для AI).
Ранее также индексировал `docker-compose.yml` и любые `*.env`.

Варианты:

**Вариант A — явные списки в `CodeLoader`** (минимальный).
Оставить `ROOT_FILES` как whitelist для корневых файлов, ввести `EXCLUDED_PATHS`
(захардкоженный чёрный список: `CLAUDE.md`, `audit.md`, `TODO.md` и т.п.). Плюсы:
просто, одно место. Минусы: изменения списка требуют правки Kotlin.

**Вариант B — `.ragignore` файл в корне репо.**
Файл `.ragignore` синтаксиса `.gitignore`. При старте `CodeLoader` и `DocumentLoader`
читают его и применяют паттерны к relative path. Плюсы: контроль со стороны автора
без правок кода, знакомый синтаксис. Минусы: ещё один файл конфигурации, придётся
аккуратно выбирать библиотеку или написать минимальный matcher.

**Вариант C — frontmatter-флаг `rag_skip: true` в документах** (только для `docs/`).
Уже есть аналог: `cv_skip: true` для CV-генератора. Можно расширить на RAG.
Плюсы: консистентно с существующим паттерном. Минусы: работает только для
markdown в `docs/`, не решает проблему с кодом.

**Рекомендация.** Вариант A как быстрая правка + Вариант C для markdown.
Вариант B — если список разрастётся, но начинать с него избыточно.

Решение за автором — ничего не реализовано, `CLAUDE.md` пока по-прежнему в
`ROOT_FILES`.
