# Инструкция агенту

Ты отвечаешь на вопрос об архитектурных решениях проекта. Тебе даны все ADR-файлы
из `docs/adr/`. Их много (20+), поэтому ответ — **каталог**, а не пересказ каждого.

Требования:

1. Группируй ADR по темам. Возможные группы (создавай сам на основе содержимого):
   - Фундаментальные (каталог проекта, домен, source of truth)
   - Frontend (фреймворк, композиция, мобильная вёрстка)
   - Backend (сервис, язык, RAG-пайплайн)
   - LLM (роль, fallback, UI-блоки, skills)
   - Инфраструктура (CI/CD, деплой)
   - Контент (стратегия чтения, CV-генератор, блог)

2. Для каждого ADR — одна строка: `- [NNN] Название — суть в 1 предложении. [N]`
   где `[N]` — индекс в списке источников.

3. Сортируй ADR внутри группы по номеру.

4. **Пропусти README.md и how-to.md** — это мета-документы, не решения.

5. **Summary** — 2-3 предложения: сколько ADR, на какие темы, общая линия
   (например, «от монорепо с декларативными артефактами к трёхслойной системе
   docs+app+server с RAG-агентом»).

6. **Detail** — группированный каталог по пункту 1.

Не цитируй содержимое ADR дословно — достаточно одно-предложного резюме каждого.
Пользователь может открыть любой ADR по клику на источник для деталей.
