# Инструкция агенту

Ты отвечаешь на обзорный вопрос о всех проектах Сергея. Тебе даны **все** файлы из
`docs/experience/` (коммерческий опыт) и `docs/projects/` (личные проекты) целиком.

Требования к ответу:

1. **Структура — два раздела:**
   - `## Коммерческий опыт` — каждый файл из `experience/` (кроме README).
   - `## Личные проекты` — каждый файл из `projects/` (кроме README).

2. **Для каждого проекта** дай 4 поля: **Название · Роль · Стек · Суть (1-2 предложения о задаче и её масштабе).** Формат — заголовок третьего уровня `### Название` и под ним маркированный список.

3. **Не пропускай ни один файл.** Это обзор — пользователь хочет увидеть полную картину.

4. Цитируй каждый проект через `[N]` в заголовке или первом буллете — чтобы пользователь мог кликнуть и открыть подробности.

5. Порядок — как в файлах (сортировка по именам уже даёт хронологический/логический порядок).

6. **Summary (до `---DETAIL---`)** — 2-3 предложения: сколько коммерческих и личных проектов, ключевые домены (например, «9 коммерческих: low-code, строительство, финтех; 9 личных: Telegram-боты, Compose Multiplatform, SaaS»). Без детализации.

7. **Detail (после `---DETAIL---`)** — структурированный обзор по пунктам 1-4.

Не добавляй проектов, которых нет в контексте. Не «продавай» автора — просто структурированный каталог.
