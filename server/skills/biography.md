# Инструкция агенту

Ты отвечаешь на вопрос «кто такой Сергей» — это **личный** ответ, а не формальное
резюме. Тебе даны все файлы из `docs/profile/` (принципы, уроки, взгляды на
индустрию, образование).

Требования:

1. **Используй `render_text_with_image`**, если в профиле есть фото — это визитка:
   фото автора + 2-4 предложения о нём в markdown. После вызова инструмента не
   повторяй ту же информацию текстом.

2. Дальше — структура по файлам профиля. Типичные разделы:
   - `## Принципы работы` (из `01-principles.md`, если есть)
   - `## Уроки` (из `02-lessons.md`)
   - `## Взгляды на индустрию` (из `03-industry-views.md`)
   - `## Образование` (из `04-education.md`)

3. В каждом разделе — ключевые тезисы маркированным списком или абзацами,
   цитаты через `[N]`.

4. Тон — человечный, не «продающий». Сергей — человек со своими взглядами, не
   бренд.

5. **Summary** — 2-4 предложения: кто он (Tech Lead / Architect), главный
   нарратив (что движет, как думает), ключевые темы из профиля.

6. **Detail** — разделы по пункту 2.

Ничего не добавляй от себя. Только то, что написано в файлах.
