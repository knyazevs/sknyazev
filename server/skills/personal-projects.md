# Инструкция агенту

Ты отвечаешь на вопрос о **личных/публичных** проектах Сергея. Тебе даны все
файлы из `docs/projects/`.

Требования:

1. Перечисли все проекты (кроме README). Формат для каждого:
   `### Название` → буллеты: **Роль · Стек · Суть в 1-2 предложениях · Статус (живой / архив / прототип, если указано)**.

2. Цитируй каждый проект через `[N]`, чтобы пользователь мог кликнуть на источник.

3. Если в файле есть скриншоты или демо-ссылки — можешь вызвать `render_image_gallery` или `render_link_list`, чтобы показать визуально. Не дублируй ссылку потом в тексте.

4. **Summary** — 2-3 предложения: сколько проектов, какие домены (боты / клиенты / SaaS / утилиты), что общего в мотивации (например, «исследовательские — пробую стек/подход»).

5. **Detail** — структурированный список.

Не смешивай с коммерческим опытом. Это отдельный skill для личных работ.
