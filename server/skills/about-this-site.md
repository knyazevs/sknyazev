# Инструкция агенту

Ты отвечаешь на мета-вопрос «что это за проект / как работает этот сайт». Тебе
даны ключевые ADR, описывающие концепцию, архитектуру и механику агента.

Требования:

1. Объясни **три слоя**: `docs/` (контент), `app/` (Astro + Svelte 5 + WebGL), `server/`
   (Kotlin/Ktor, RAG-агент).

2. Расскажи про **RAG-пайплайн**: HyDE → dense + BM25 → RRF → cosine-filter (ADR-019),
   плюс **skills-слой** поверх (ADR-026) для обзорных запросов.

3. Упомяни, что сервер работает **на Kotlin/Native** (ADR-021) — не JVM.

4. Упомяни **UI-блоки** как механизм визуальных ответов (ADR-024, ADR-025).

5. Расскажи про **мотивацию** — это личный проект-портфолио, демонстрирующий
   мышление через ADR, блог и код.

6. Summary (до `---DETAIL---`) — 2-3 предложения: что за сайт, ключевая идея,
   главные технические фишки.

7. Detail — структурированное описание: слои → RAG → skills → UI-блоки →
   мотивация.

Цитируй конкретные ADR через `[N]`. Не превращай ответ в энциклопедию —
ограничивайся фактами из ADR, пользователь может открыть каждый по клику.
